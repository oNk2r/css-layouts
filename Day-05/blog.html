How CSS Grid changed the way I think about layouts

May 9, 2026  ·  4 min read  ·  Day 5 of #csslayouts

Before Grid, building a two-column layout meant floats, clearfixes, and a lot of frustration. Today I built the same layout in three lines of CSS and it actually makes sense.

The three lines that do everything

The entire layout — wide content on the left, sidebar fixed on the right — comes down to this:

.blog-layout {display: grid;grid-template-columns: 1fr 320px; gap: 3rem;}

1fr means "take all the remaining space after the sidebar's 320px is reserved." The browser does the math. You never touch a percentage or calculate a width by hand.

Flexbox vs Grid — when to use which

After five days of exercises I have a simple rule: use Flexbox for components (navbars, card rows, button groups), use Grid for page structure (the skeleton that holds everything together).

Notice the sidebar's recent posts widget uses Flexbox to align the thumbnail and text side by side. Grid for the outer layout, Flexbox for the inner component — they work together perfectly.

The sticky sidebar trick

Adding position: sticky; top: 88px to the sidebar makes it stay visible as you scroll through a long article. This only works because align-items: start is set on the grid container — without it, the sidebar stretches to match the full height of the content column and sticky has nothing to stick inside.

What's next

Day 6 is combining everything — Grid for the full page skeleton, Flexbox for all the components inside it. That's the pattern used in almost every real website.
